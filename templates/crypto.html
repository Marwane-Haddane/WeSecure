CTF Guide: Understanding the Algorithms

🔐 Symmetric Encryption

Same key decrypts and encrypts. Fast and effective.

  • Fernet: A secure wrapper around AES-128-CBC.
    How to spot: Outputs are Base64 strings that *always* begin with gAAAAAB...
  • AES-GCM: Advanced Encryption Standard in Galois/Counter Mode. Provides both confidentiality and authenticity.
    How to spot: Looks like standard Base64 blobs. If you decode it into bytes, the first 12 bytes are the Nonce, followed by ciphertext and an auth tag.

🔑 Asymmetric Encryption

Requires Public Key to encrypt, Private Key to decrypt.

  • RSA: Standard for transmitting shared secrets.
    How to spot: You will usually find RSA Keys wrapped in PEM format, clearly marked: -----BEGIN PUBLIC KEY-----. Ciphertext is very large and appears totally random.

🔏 Hashing

One-way functions. Cannot be decrypted, only cracked.

  • SHA-256: Standard integrity checker.
    How to spot: Exactly 64 characters long in hexadecimal format (e.g., 8d969eef6...).
  • PBKDF2: Key Derivation Function. Uses a salt and thousands of iterations to make brute-forcing passwords incredibly slow.

🔄 Encoding

NOT encryption! Used to make binary data text-safe.

  • Base64:
    How to spot: Alphanumeric character set (A-Z, a-z, 0-9, +, /). It very often ends in padding characters: = or ==.
